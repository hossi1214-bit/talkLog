# talkLog マーケティング・料金設計・収益モデル 初期版

## 1. コンセプト

### ビジョン

talkLogは単なるAI英語学習アプリではない。

**自分の声で成長を記録する音声ログアプリ**

を目指す。

### コアバリュー

- AIで添削して終わりではない
- 自分の音声が学習資産になる
- 1週間前、1か月前、半年前の自分と比較できる
- 継続するほど価値が積み上がる

## 2. 連携サービス

| サービス | 用途 | ランニングコスト |
|---|---|---|
| Google Play | Android公開 | 初回登録費のみ |
| Apple Store | iOS公開 | 年会費 約99USD/年 |
| OpenAI API | 音声認識・AI添削・翻訳・分析 | 利用量に応じて増加。最大コスト |
| Supabase | 認証・DB・Storage・Edge Functions | Free → Pro $25/月〜 |

最も重要なのは OpenAI API コスト。Google Play、Apple、Supabaseは固定費に近く、利益を左右するのはAI利用量となる。

## 3. OpenAI APIコスト予測

| 利用状況 | 1ユーザー/月 |
|---|---:|
| ライト 5分/日 | 約50〜100円 |
| 標準 10〜15分/日 | 約100〜250円 |
| ヘビー 30分/日 | 約300〜500円 |

運営開始後は、ユーザーごとのAI利用量とAPIコストを記録し、無料プラン・Premium料金を継続的に最適化する。

## 4. 音声データ方針

音声は学習履歴であり、ユーザーの資産。添削後も削除しない。

### 保存仕様

- 録音形式: AAC m4a
- 音質: 64kbps
- 保存先: Supabase Storage
- 自動削除: なし
- ユーザー自身で削除可能

## 6. 音声保存容量

Free は 200MB。

64kbps AACの場合:

| 利用量 | 保存期間の目安 |
|---|---|
| 5分/日 | 約2〜3か月 |
| 10分/日 | 約40日 |
| 15分/日 | 約1か月 |

無料体験として十分な容量を提供する。

## 7. 容量メーター

ホーム画面に表示する。

表示例:

```text
音声ストレージ
82MB / 200MB 使用中
残り18MBです。
Premiumなら容量を気にせず保存できます。
```

目的:

- 現在の使用量が分かる
- 保存容量不足を事前に把握できる
- Premiumへの自然な導線になる

## 8. 広告方針

通常広告は導入しない。

対象外:

- バナー広告
- 起動時広告
- 全画面広告
- 学習途中広告

理由は学習体験を最優先するため。

### リワード広告

ユーザーが任意で広告を見ることで AI添削を1回追加できる。

Freeプラン:

- AI添削: 1日5回
- 広告を見る: +1回
- 最大3回/日
- 最大8回/日利用可能

表示タイミング:

- 本日の無料添削終了後

表示しないタイミング:

- アプリ起動時
- 録音開始前
- 添削結果表示前

## 12. マーケティングメッセージ

### コンセプト

昨日の自分より話せる。

### 提供価値

talkLogは、英語を学ぶアプリではなく、英語で話した自分を記録し、成長を実感できるアプリである。

学習を積み重ねるほど、自分だけの音声ログが増え、英語力の成長を耳で実感できる。それがtalkLog最大の価値である。
